## What changed from run-003

Root cause of run-002/003 empty captures: **timing**. The matmul completed in under 1 second (PyTorch import
overhead consumed ~18s of the 19s pod lifetime), and the Nsight coordinator's GO signal arrived after the
CUDA work was already done (START_TRIGGER: Manual, coordinator round-trip ~19s).

Fix applied:
- Added 60s pre-CUDA sleep (gives nsys time to initialize and receive the coordinator GO signal)
- Changed to a 180s CUDA loop (wide capture window, ≫ coordinator round-trip)
- Added 60s post-CUDA sleep (nsys flush + MinIO upload before exit)
- Started Nsight collection AFTER `main` container was Running (not before deploy)
