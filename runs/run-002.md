## Root cause of empty capture

`META_DATA_CAPTURE` in SQLite: `PROCESS_0:COLLECT_CUDA_TRACE=false`  
`DIAGNOSTIC_EVENT`: "No CUDA events collected. Does the process use CUDA?"

Two compounding causes:

1. **`NVIDIA_VISIBLE_DEVICES=void`** set by the process hook on the nsys-relaunched python3
   as an anti-recursion guard. CUPTI reads this env var to find GPU contexts — `void`
   prevents CUPTI from initializing, so nsys disables CUDA trace for that process.

2. **No `SYS_PTRACE` / restrictive seccomp** — KFP component pods drop ALL capabilities
   (`securityContext.capabilities.drop=[ALL]`) and use `seccompProfile: RuntimeDefault`.
   `RuntimeDefault` blocks `perf_event_open` which nsys uses for CPU sampling.
   Diagnostic: "Unable to configure the collection of CPU IP/backtrace samples."

The python3 computation itself succeeded (`Result norm: 741369.4375`) — CUDA works fine
at the application level, just not at the CUPTI tracing layer.

Also: `nsys exit code = 139` (SIGSEGV) during report finalization, but the `.nsys-rep`
was already uploaded before the crash.

## Nsight Operator REST API (discovered this session)

The coordinator exposes a REST API at port 80 (host port-forward: 9097):

| Endpoint | Purpose |
|---|---|
| `GET /api/v1/sessions/` | List active sessions |
| `POST /api/v1/sessions/` | Create new session |
| `DELETE /api/v1/sessions/{sid}` | End session |
| `POST /api/v1/sessions/{sid}/collect` | Start collection |
| `DELETE /api/v1/sessions/{sid}/collect` | Stop collection |
| `GET /api/v1/sessions/{sid}/files` | List captured files (UUIDs + filenames) |
| `GET /api/v1/tags/` | List service tags (registered pods) |

MinIO: bucket `nsight-reports`, path `{uuid}/{filename}.nsys-rep`  
Credentials: secret `nsight-operator-cloud-storage-minio-credentials`

## Correct workflow for run-003

1. Open Nsight UI at http://localhost:8889 → create a session
2. `POST /api/v1/sessions/{sid}/collect` — start collection
3. Submit KFP run
4. Wait for pod to complete (main exits 0)
5. Wait ~15s for OTLP grace period
6. `DELETE /api/v1/sessions/{sid}/collect` — stop collection
7. `GET /api/v1/sessions/{sid}/files` — get file UUID and filename
8. Download: `GET http://localhost:9099/nsight-reports/{uuid}/{filename}` with AWS SigV4 auth
9. Run `/nsight-interpret run-003`
