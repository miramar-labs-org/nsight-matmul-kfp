## Analysis

**Why 166 > 27?**

Both runs had `STOP_TRIGGER=Delayed` and both had the 60s post-CUDA sleep. The 6x improvement
in captured kernel records comes from the 60s sleep itself: the hardware trace flush timer
(`--cuda-flush-interval=100ms`) continues running during the sleep, committing more GPU timestamp
data from the hardware counter ring buffer. run-009 may have had a smaller flush window in
practice, or the buffer state varied.

**Dropped events formula:**
- Total kernel launches: 6,261 (cuLaunchKernel calls)
- Captured: 166
- Dropped: 6,096
- 166 + 6,096 = 6,262 ≈ 6,261 ✓

The hardware trace IS recording all kernel events, but 6,096 GPU timestamps remain
incomplete at process exit — they haven't been drained from the GPU driver's ring buffer.

**Root cause of remaining drops:** With `STOP_TRIGGER=Delayed`, nsys stops collection
when the python3 process exits. At that point, the hardware counter ring buffer still holds
unread events. nsys drains what it can, but the GPU driver doesn't guarantee all timestamps
are available immediately after process exit.

**What DELETE /collect during sleep would do:** Sending STOP before the process exits changes
`STOP_TRIGGER` from Delayed to something earlier, giving nsys an explicit stop signal with the
process still alive. nsys can then actively drain the GPU ring buffer while the Python process
is still in the 60s sleep — a much larger flush window than at process exit.

## Monitor script bug

The pod detection logic ran:
```bash
kubectl get pods -n kubeflow | grep "nsight-matmul-kfp" | grep -v "driver" | head -1
```
The old pod (`6dczm`) was still listed (otlp-proxy container still running) and appeared first.
The new pod (`7xrrt`) was not yet in Running state when the check ran.

**Fix for run-011:** Filter by READY status or check that the pod suffix is different from
the previous run:
```bash
# Wait for a new pod (suffix differs from last run)
PREV_SUFFIX="6dczm"
kubectl get pods -n kubeflow | grep "nsight-matmul-kfp" | grep -v "driver\|$PREV_SUFFIX" | head -1
```
